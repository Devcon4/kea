# Shared Package Instructions

## Purpose

`@kea/shared` contains types, validation schemas, and utilities shared across:

- `@kea/agent` — consumes types and Result utilities
- `@kea/api` — consumes types, schemas, and Result utilities
- `@kea/dashboard` — consumes types (via API responses)

## What Belongs Here

- **Domain types** — `Session`, `SitemapEntry`, `Finding`, `PageStatus`, `Severity`, etc.
- **Zod schemas** — Request validation schemas used by the API and potentially the agent client.
- **Result type** — `Ok`, `Err`, `tryCatch`, `tryCatchSync`, `unwrap`, `mapResult`.
- **Pure utility functions** — Things that depend on no runtime (no Node.js, no DOM).

## What Does NOT Belong Here

- Database schemas (drizzle tables) — those live in `@kea/api`.
- UI components — those live in `@kea/dashboard`.
- Agent logic — that lives in `@kea/agent`.
- Anything with side effects (I/O, network, filesystem).

## Exports

- `@kea/shared` — re-exports everything
- `@kea/shared/types` — domain types only
- `@kea/shared/result` — Result type and utilities
- `@kea/shared/schemas` — Zod validation schemas

## Testing

- Vitest, colocated: `foo.ts` → `foo.spec.ts`.
- Test schemas with `.safeParse()` — valid and invalid inputs.
- Test Result utilities with sync and async scenarios.
