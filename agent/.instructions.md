# Agent Worker Instructions

## Module Structure

The agent worker is a TypeScript Node.js app. Each module is a domain:

- `src/a2a/` — A2A protocol types, server, client. Each agent is an A2A server.
- `src/llm/` — OpenAI-compatible LLM client. Thin wrapper around `openai` npm package.
- `src/browser/` — Stagehand browser automation. Exposes actions as A2A-callable tools.
- `src/memory/` — SQLite persistence for sitemap and findings.
- `src/agents/` — Agent role definitions (coordinator, navigator, tester). Each agent has a system prompt and tool set.
- `src/server/` — Hono HTTP server that mounts A2A endpoints and health checks.

## Import Rules

- Import directly from source files: `import { chat } from "./llm/client.js";`
- Do NOT create barrel `index.ts` files.
- Use `.js` extensions in imports (ESM).

## Error Handling

- All functions that can fail return `Result<T, E>`.
- Use `tryCatch()` / `tryCatchSync()` from `src/result.ts` to wrap external calls.
- Never use try/catch directly in business logic.

## Agent Architecture

- Each agent role is an A2A server with its own Agent Card.
- Agents communicate via A2A `sendMessage`. The coordinator delegates to navigator/tester.
- LLM calls happen inside the agent's task handler — receive A2A message → build prompt → call LLM → execute tools → return A2A response.
- Browser tools are Stagehand actions exposed as OpenAI function-calling tools.

## Testing

- Vitest, colocated test files (`*.test.ts`).
- Mock Ollama/vLLM responses in tests — never hit a real LLM in unit tests.
- Use `better-sqlite3` in-memory mode (`:memory:`) for store tests.
