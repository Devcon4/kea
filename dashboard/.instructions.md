# Dashboard Instructions

## Framework

- **Lit** web components with TypeScript strict mode.
- **Vite** for dev server and production builds.
- Components are custom elements registered via `@customElement("kea-*")` — all elements use the `kea-` prefix.

## State Management

- **Services** hold application state in RxJS `BehaviorSubject`s. Services are plain TypeScript classes — no Lit dependency.
- **Components** convert observables to signals via `toSignal()` from `src/rxjs-interop.ts` for rendering.
- **`@lit/context`** provides DI — app shell `@provide()`s services, child components `@consume()` them.
- Context keys are defined in `src/contexts.ts`.

## Component Patterns

- Extend `SignalWatcher(LitElement)` as base class for auto-tracking signals in `render()`.
- Use `@property()` for public attributes, `@state()` for internal reactive state, and signals (via `toSignal()`) for service-derived presentation state.
- Styles use `css` tagged template literals — no external CSS files per component.
- Prefer composition over inheritance for component behavior.

## File Layout

```
src/
  app.ts                   # Shell component, provides services
  contexts.ts              # @lit/context keys
  rxjs-interop.ts          # toSignal / toObservable bridge
  services/
    session.service.ts     # RxJS-based state service
  components/
    session-list.ts        # Feature components
  theme.css                # Global reset + CSS custom properties
```

## Testing

- **Vitest** with colocated test files: `foo.ts` → `foo.spec.ts`.
- Use `jsdom` or `happy-dom` environment for component tests.
- Mock services by providing test instances via `@lit/context` in a test host element.

## Naming

- Custom element tags: `kea-app`, `kea-session-list`, `kea-session-card`, etc.
- Service classes: `SessionService`, `FindingsService`, etc.
- Context keys: `sessionServiceContext`, `findingsServiceContext`, etc.
