# API Service Instructions

## Framework

- **Hono** HTTP framework with Node.js adapter (`@hono/node-server`).
- **drizzle-orm** with **PostgreSQL** via `postgres` (postgres.js driver).
- **Zod** schemas from `@kea/shared` for request validation.

## Module Structure

- `src/db/schema.ts` — Drizzle table definitions (sessions, sitemap, findings).
- `src/db/connection.ts` — Database connection factory.
- `src/routes.ts` — All HTTP route handlers.
- `src/index.ts` — Server entry point, shutdown handling.
- `src/logger.ts` — Pino structured logging.

## API Design

- RESTful routes scoped under `/api/sessions/:id/`.
- All sitemap and findings operations are session-scoped.
- Zod validation on all POST/PUT/PATCH bodies — return 400 with issues on failure.
- Use drizzle's `.returning()` for insert/update responses.
- Health endpoints: `GET /healthz`, `GET /readyz`.

## Error Handling

- Route handlers return JSON error objects with appropriate HTTP status codes.
- No try/catch in route handlers — let Hono's error handling middleware catch unhandled errors.
- Use `@kea/shared` Result types in internal logic.

## Testing

- Integration tests against a real PostgreSQL instance (set `TEST_DATABASE_URL`).
- Tests use Hono's `app.request()` — no HTTP server needed.
- Colocated test files: `foo.ts` → `foo.spec.ts`.

## Database

- Migrations managed by `drizzle-kit` (`npm run db:generate`, `npm run db:migrate`).
- Connection string from `DATABASE_URL` env var.
- Default: `postgres://kea:kea@localhost:5432/kea`.
