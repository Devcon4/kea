# Kea — Coding Standards

## Language & Style

- **TypeScript strict mode** — `strict: true` in tsconfig, no `any` unless absolutely unavoidable.
- **`type` over `interface`** — always use `type` aliases. Never use `interface`.
- **Prettier defaults** — double quotes, semicolons, 2-space indent, trailing commas.
- **Full imports** — no barrel files (index.ts re-exports). Import directly from the source file. Use TypeScript path aliases or import maps where tooling supports it.

## Patterns

- **Result types** — use `Result<T, E>` for error handling. Never throw exceptions in business logic. Only throw at system boundaries (startup, unrecoverable). Import from `src/result.ts`.
- **Early return** — always use guard clauses. Return early on error/edge cases. The happy path should be the least-indented code.
- **Max 3 levels of nesting** — if code is deeper than 3 blocks, refactor. Extract functions, use early returns, or restructure.
- **No else / else-if** — use early returns, lookup maps, ternaries, or switch expressions instead. `else` blocks are banned.
- **Functional patterns** — prefer `map`, `filter`, `reduce`, pure functions, immutable data. Go-style simplicity: explicit, readable, verbose is fine.
- **Reactive patterns** — prefer RxJS Observables and reactive pipelines for state management, event streams, and async flows. Use `Subject`, `BehaviorSubject`, and operators (`map`, `filter`, `scan`, `switchMap`, etc.) over mutable arrays, event emitters, or manual state mutation. State should flow through pipelines, not be imperatively modified.
- **Temporal API** — use `Temporal.Now.instant()` and `Temporal.Instant` over `Date` and `Date.now()` where the runtime supports it. Fall back to `Date` only when Temporal is not available.

## Typing

- **Strictly type everything** — no implicit `any`, no untyped parameters, no `as` casts unless unsafe boundary (e.g., JSON parse). Build complex type helpers and utility types when needed.
- **Branded/nominal types** where IDs or domain values need to be distinguished (e.g., `type TaskId = string & { readonly __brand: "TaskId" }`).
- **Discriminated unions** for state machines and message types.

## Architecture

- **DDD** — organize by domain (agents, browser, memory, a2a), not by technical layer. Domain types live in their module.
- **TDD** — write tests alongside implementation. Use Vitest. Test files are colocated: `foo.ts` → `foo.test.ts`.
- **A2A protocol** — agent-to-agent communication uses Google's A2A protocol. Each agent is an A2A server with an Agent Card.
- **OpenAI-compatible API** — LLM calls go through the `openai` npm package. Backend is configurable (vLLM, Ollama, etc.) via `LLM_BASE_URL` env var.

## Logging

- **pino** for structured JSON logging. Create child loggers per module via `createLogger("module-name")`.
- Log levels: `error` for failures, `warn` for degraded, `info` for lifecycle events, `debug` for detailed flow.

## Naming

- **Files**: kebab-case (`agent-card.ts`, `a2a-server.ts`)
- **Types**: PascalCase (`TaskState`, `AgentCard`)
- **Functions/variables**: camelCase (`sendMessage`, `getAgentCard`)
- **Constants**: SCREAMING_SNAKE_CASE for true constants (`MAX_RETRIES`, `DEFAULT_MODEL`)
